resenDev

Mock the Resend API locally. Capture, inspect, and debug transactional emails — without sending a single one.

$ docker run -p 3099:3099 jupitercl/resendev

Why resenDev

Everything you need,
nothing you don't.

🔀
Drop-in replacement
Just set an env var. No code changes to your app — every Resend SDK picks it up automatically.
🔍
Full inspection
HTML preview, plain text, raw source, headers, attachments, and JSON — all in one place.
⚡
Real-time updates
Emails appear instantly via Server-Sent Events. No polling, no page refresh needed.
🔎
Full-text search
Search across subject, body, sender, and recipient across all captured emails instantly.

🌙
Dark mode
System-aware dark/light mode. Plus error simulation and delay settings for realistic testing.

Quick Start

Up and running
in 30 seconds.

Run the container, point your SDK at localhost:3099, and start sending. Any API key is accepted — the header just needs to be present.

docker compose 
services:
  resendev:
    image: jupitercl/resendev:latest
    ports:
      - "3099:3099"
    volumes:
      - resendev_data:/app/data

volumes:
  resendev_data:
Node.js / TypeScript 
# .env.development
RESEND_BASE_URL=http://localhost:3099
RESEND_API_KEY=re_any_key_works
TypeScript 
import { Resend } from "resend";

const resend = new Resend("re_any_key");

await resend.emails.send({
  from:    "no-reply@myapp.com",
  to:      "user@example.com",
  subject: "Hello!",
  html:    "<h1>Hi!</h1>",
});

Set the env var and you're done — no code changes required.
